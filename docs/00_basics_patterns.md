# 유형 00 — Lv.0 문법 패턴

> **대상**: 문법(`for`/`if`/리스트/문자열)은 아는데, "문제를 보면 어떻게 코드로 옮길지"가 막히는 단계.
> **목표**: 자료구조·알고리즘 없이, **문제 지문 → 코드 구조 번역**을 손에 익히기.

핵심은 새 문법을 배우는 게 아니라, **"이런 말이 나오면 이런 코드"**라는 패턴 매칭을 몸에 새기는 것입니다.

---

## 패턴 1 — "모든 ~에 대해 무언가 한다" → 반복문

지문에 **"각각", "모든", "1부터 n까지"**가 나오면 `for`입니다.

```python
# "1부터 n까지의 합을 구하라"
def solution(n):
    total = 0
    for i in range(1, n + 1):   # 1, 2, ..., n
        total += i
    return total
```

> 번역 규칙: **"~까지" → `range`**, **"누적/합/개수" → 루프 밖에 변수 만들고 루프 안에서 갱신**.

---

## 패턴 2 — "조건을 만족하는 것만" → 반복 + if

**"~인 경우만", "짝수만", "3보다 큰"** → 루프 안에 `if`.

```python
# "리스트에서 짝수만 더하라"
def solution(numbers):
    total = 0
    for x in numbers:
        if x % 2 == 0:          # 짝수 조건
            total += x
    return total
```

같은 걸 한 줄로 (익숙해지면):

```python
def solution(numbers):
    return sum(x for x in numbers if x % 2 == 0)
```

> 가르칠 포인트: 먼저 **풀어쓴 for+if**로 이해 → 그 다음 **컴프리헨션**으로 압축. 순서가 중요합니다.

---

## 패턴 3 — "개수를 세라 / 최대·최소" → 누적 변수

```python
# "양수의 개수를 세라"
def solution(numbers):
    count = 0
    for x in numbers:
        if x > 0:
            count += 1
    return count
```

```python
# "가장 큰 값을 찾아라" (max를 안 쓰고 원리 이해)
def solution(numbers):
    biggest = numbers[0]            # 일단 첫 값을 후보로
    for x in numbers:
        if x > biggest:             # 더 크면 후보 교체
            biggest = x
    return biggest
```

> 실무에선 `len([x for x in numbers if x > 0])`, `max(numbers)`로 줄이지만, **원리를 먼저** 보여주는 게 응용력에 좋습니다.

---

## 패턴 4 — 문자열 다루기

Lv.0의 절반은 문자열입니다. 자주 쓰는 도구:

```python
s = "hello world"
s.upper()            # "HELLO WORLD"
s.split()            # ['hello', 'world']  (공백 기준)
s.replace("l", "")   # "heo word"
len(s)               # 11
s[0], s[-1]          # 'h', 'd'   (인덱싱)
s[0:5]               # 'hello'    (슬라이싱)
"-".join(["a", "b"]) # "a-b"
```

```python
# "문자열에서 모음의 개수를 세라"
def solution(s):
    count = 0
    for ch in s:
        if ch in "aeiou":     # 'in'으로 멤버십 검사
            count += 1
    return count
```

---

## 패턴 5 — "각 자리/각 글자를 변환" → 순회하며 새 리스트

```python
# "각 숫자를 제곱한 리스트를 반환하라"
def solution(numbers):
    result = []
    for x in numbers:
        result.append(x * x)
    return result
```

압축형:

```python
def solution(numbers):
    return [x * x for x in numbers]
```

> **"입력 리스트 → 같은 길이의 변환된 리스트"** 모양이면 거의 다 이 패턴입니다.

---

## 지문 → 코드 번역 치트시트

| 지문에 이런 말이 나오면 | 이렇게 시작 |
|------------------------|------------|
| "각각, 모든, 1부터 n까지" | `for ... in range(...)` |
| "~인 것만, ~인 경우" | 루프 안에 `if 조건` |
| "개수, 합, 누적" | 루프 밖 `cnt=0` / `total=0` 만들고 갱신 |
| "가장 큰/작은" | 후보 변수 두고 비교 교체 (또는 `max`/`min`) |
| "문자열을 나눠/바꿔" | `split`, `replace`, 슬라이싱 |
| "각 원소를 변환한 리스트" | 새 리스트에 `append` (또는 컴프리헨션) |

---

## 세션 진행 제안

이 패턴을 깐 다음 **프로그래머스 Lv.0 문제 3~4개를 같이 푸는** 방식이 효과적입니다. 예를 들어:

- 두 정수 사이의 합
- 짝수의 합
- 문자열 뒤집기
- 자릿수 더하기

각 문제를 **"치트시트 어느 줄에 해당하나?"**로 분류 → 패턴대로 코드 작성. 이 흐름을 반복하면 응용력이 붙습니다.

---

## 예제 풀이

각 문제를 **① 지문에서 신호 찾기 → ② 치트시트로 패턴 분류 → ③ 코드 작성** 순서로 풉니다.

### 예제 1 — 두 정수 사이의 합

> 두 정수 `a`, `b`가 주어질 때, `a`와 `b` 사이에 속한 모든 정수의 합을 구하라. (`a ≤ b`가 아닐 수도 있음)

**① 신호**: "사이에 속한 모든 정수", "합"
**② 패턴**: 패턴 1 (모든 ~에 대해 → 반복) + 패턴 3 (합 → 누적 변수)
**③ 작성**:

```python
def solution(a, b):
    # a가 b보다 클 수도 있으니 작은 값/큰 값을 정리
    if a > b:
        a, b = b, a
    total = 0
    for i in range(a, b + 1):   # a부터 b까지 (b 포함이라 +1)
        total += i
    return total
```

> 함정 포인트: **`a > b`인 경우**를 빠뜨리면 `range`가 빈 범위가 되어 0이 나옴. "순서가 보장 안 된다"는 한 줄을 놓치지 않는 훈련.
>
> 공식을 알면 한 줄: `return (a + b) * (abs(b - a) + 1) // 2` — 하지만 처음엔 루프로 원리부터.

---

### 예제 2 — 짝수의 합

> 정수 `n`이 주어질 때, `n` 이하의 짝수를 모두 더한 값을 반환하라.

**① 신호**: "n 이하의", "짝수를", "모두 더한"
**② 패턴**: 패턴 1 (반복) + 패턴 2 (조건만) + 패턴 3 (합)
**③ 작성**:

```python
def solution(n):
    total = 0
    for i in range(1, n + 1):
        if i % 2 == 0:          # 짝수만
            total += i
    return total
```

> 변형 연습: `range(2, n + 1, 2)`로 처음부터 짝수만 순회하면 `if`가 사라짐. "조건으로 거르기 vs 애초에 그것만 만들기" 두 사고방식을 비교해 보세요.

---

### 예제 3 — 문자열 뒤집기

> 문자열 `s`가 주어질 때, 순서를 뒤집은 문자열을 반환하라.

**① 신호**: "문자열을", "뒤집은"
**② 패턴**: 패턴 4 (문자열 다루기) + 패턴 5 (각 글자를 순회)
**③ 작성** — 원리부터:

```python
def solution(s):
    result = ""
    for ch in s:
        result = ch + result    # 새 글자를 '앞에' 붙이면 역순이 됨
    return result
```

> 슬라이싱을 알면 한 줄: `return s[::-1]` (`[::-1]`은 "끝에서부터 거꾸로"). 원리(앞에 붙이기 누적)를 먼저 이해한 뒤 슬라이싱으로 압축하는 순서가 좋습니다.

---

### 예제 4 — 자릿수 더하기

> 정수 `n`이 주어질 때, 각 자리 숫자의 합을 구하라. (예: `123` → `1 + 2 + 3 = 6`)

**① 신호**: "각 자리 숫자의", "합"
**② 패턴**: 패턴 4 (문자열로 변환) + 패턴 1·3 (순회하며 합)
**③ 작성** — 문자열로 바꿔 한 글자씩:

```python
def solution(n):
    total = 0
    for ch in str(n):           # 숫자를 문자열로 → 한 글자씩 순회
        total += int(ch)        # 글자를 다시 정수로 바꿔 더함
    return total
```

> 핵심 아이디어: **"각 자리"가 막막하면 `str(n)`으로 바꿔서 글자 순회로 바꾼다.** 숫자 ↔ 문자열 변환은 Lv.0의 단골 무기.
>
> 수학으로 푸는 법(나머지 연산)도 있지만, 문자열 변환이 더 직관적입니다:
> ```python
> def solution(n):
>     total = 0
>     while n > 0:
>         total += n % 10        # 마지막 자리
>         n //= 10               # 마지막 자리 버리기
>     return total
> ```

## 정리

네 문제 모두 **새 문법이 아니라 패턴 조합**으로 풀렸습니다:

| 예제 | 쓰인 패턴 |
|------|----------|
| 두 정수 사이의 합 | 반복 + 누적 (+ 경계 조건 처리) |
| 짝수의 합 | 반복 + 조건 + 누적 |
| 문자열 뒤집기 | 문자열 순회 + 누적 |
| 자릿수 더하기 | 형 변환(`str`/`int`) + 순회 + 누적 |

> 응용력의 핵심은 **"지문의 단어 → 치트시트의 줄 → 코드 골격"** 변환을 빠르게 하는 것. 문제를 풀 때마다 이 3단계를 소리 내어 짚는 습관이 가장 효과적입니다.
